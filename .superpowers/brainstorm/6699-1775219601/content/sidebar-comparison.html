Key Differences

Current
  • Single "New Task" button opens modal
  • 6+ fields to fill (description, prompt, mode, directory...)
  • Tasks grouped by path as afterthought
  • Remote/local toggle adds complexity
Proposed
  • Repos are first-class sidebar items
  • "+" per repo → instant task creation
  • Zero fields — worktree + terminal auto-created
  • Description auto-set from first prompt via LLM
